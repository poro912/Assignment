서비스, 함수, 루틴
윈도우 API함수 :윈도우 API에 문서화돼 있고 호출 가능한 서브 루틴
네이티브 시스템 서비스 (시스템호출)
	운영체제 하부에 문서화되지 않은 서비스
	유저모드에서 호출 가능하다
커널 지원 함수(루틴) :커널 모드에서만 호출 가능한 운영체제 내부의 서브루틴
윈도우 서비스	
	서비스 컨트롤 관리자에 의해 시작된 프로세스
	유저 모드 프로세스에서 동작한다.
DLL
	호출 가능한 서브루틴이 모여있는 바이너리
	애플리케이션에 의해 동적으로 로드될 수 있다.
	애플리케이션간에 DLL을 공유할 수 있다.
	참조중인 DLL의 코드가 메모리상의 하나의 복사본만 존재한다.

프로세스 스레드 잡
프로그램 :명령의 정적 연속체
프로세스 :프로그램의 인스턴스가 실행될 때 사용되는 리소스의 집합을 위한 컨테이너

프로세스의 구성요소
	전용 가상 주소공간 : 프로세스가 사용할 수 있는 가상 메모리 주소의 집합
	실행 가능한 프로그램 : 초기 코드와 데이터를 정의
			프로세스의 가상 주소 공간에 매핑됨
	세마포어와 통신 포트, 파일 등의시스템 리소스 핸들
	엑세스 토큰 : 연관된 사용자, 보안 그룹, 특원, 사용자 계정 컨트롤 가상화 상태, 세션, 제한된 사용자 계성 상태를 식별한다.
	프로세스 ID : 고유 식별자
	최소 한 개의 실행 스레드

스레드의 컴포넌트
	프로세서의 상태를 표현하는 CPU 레지스터의 집합
	두 개의 스택
		1. 커널 모드에서 실행될 때 사용된다.
		2. 유저 모드에서 실행될 때 사용된다.
	스레드 로컬 스토리지 전용 저장영역, 런타임 라이브러리, DLL
		서브시스템이 사용하기 위함
	스레드 ID
	보안 컨텍스트(토큰)
		멀티스레드 서버 애플리케이션에서 사용된다.
		서버 프로그램이 서비스하는 클라이언트의 보안 컨텍스트 자격을 변경하기 위해 사용
스레드 컨텍스트
	휘발성 레지스터와 스택 전용저장영역
	아키텍쳐 한정적이다
	GetTHreadContext 함수를 사용해 정보에 접근할 수 있다.


파이버
	윈도우에 내장된 우선순위 기반스케줄링 대신 자신의 실행 스레드를 스케줄링 하는 것을 허용
	경량 스레드 라고도 불린다.
	스케줄링 관점에서 파이버는 커널에 보이지 않는다.
	유저모드인 kernel32.dll에 구현되어있다.
	ConvertThreadToFiber()를 호출하여 파이버로 변환한다.
	이후 CreateFiber()로 추가적인 파이버를 생성할 수 있다.
	SwitchToFiber() 함수를 호출해 수동으로 선택되지 전까지는 실행되지 않는다.

UMS(유저모드 스케줄러 스레드)
	파이버의 단점을 없애고 기본적인 장점을 제공한다.
	자신의 커널 스레드 상태를 가지고 있어 커널에 보인다.
	두개 이상의 UMS스레드가 작업을 필요로 하면 스케줄러 개입 없이 컨텍스트를 전환할 수 있다.
	컨텍스트 전환은 유저모드에서 이뤄저 커널관점에서는 동일한 커널 스레드로 인식된다.
	(5장 프로세스, 스레드, 잡 참조)
